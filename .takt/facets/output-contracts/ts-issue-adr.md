# ADR（Architecture Decision Record）出力フォーマット（ts-issue-adr）

以下の形式で意思決定ログを出力してください。曖昧点がない場合も空ファイルとして出力してください。

---

# ADR: Issue #{番号} 意思決定ログ

曖昧点がない場合は、以下を記述してください：

> Issue の要件に曖昧点はありませんでした。ADR の作成は不要です。

曖昧点がある場合は、以下の形式で各意思決定を記述してください：

---

## ADR-001: {意思決定のタイトル}

### 背景（Background / Context）

[なぜこの意思決定が必要だったか。Issue の記述のどの部分が曖昧だったかを説明]

### 選択肢（Options / Alternatives）

1. **選択肢 A**: [選択肢の説明]
   - メリット: [利点]
   - デメリット: [欠点]

2. **選択肢 B**: [選択肢の説明]
   - メリット: [利点]
   - デメリット: [欠点]

### 決定（Decision / Rationale）

**選択: 選択肢 A**

[選んだ理由を記述。なぜこの選択肢が最適かを説明する]

### 影響

[この決定が実装・テスト戦略に与える影響を記述]

---

## 出力ルール

1. ADR は reports/ ディレクトリにのみ保存する（プロジェクトリポジトリには残さない）
2. 1つの曖昧点 = 1つの ADR エントリ（ADR-001, ADR-002, ...）
3. 背景・選択肢・決定の3セクションは必須
4. 決定は明確に1つに絞ること（「場合によって異なる」は禁止）
