## 出力ルール

1. **判定**
   - 指摘事項がない、またはすべてLowの場合: `APPROVED`
   - Critical または High の指摘がある場合: `NEEDS_FIX`

2. **サマリー**
   - 各重要度の件数を正確にカウント
   - 重要度内訳は必ず記載

3. **指摘事項**
   - 重要度の高い順（Critical → High → Medium → Low）でソート
   - 各指摘事項には必ず以下を含める：
     - ファイルパスと行番号
     - 明確なタイトル
     - 重要度ラベル
     - カテゴリ分類
     - 詳細説明（問題の内容と影響）
     - 推奨対応（具体的な修正方法）

4. **重要度の基準**
   - **Critical**: 致命的な問題（動作に影響、セキュリティリスク）
   - **High**: 重要な問題（保守性の大幅な低下、パフォーマンス問題）
   - **Medium**: 改善推奨（コード品質の向上）
   - **Low**: マイナーな指摘（スタイルの統一等）

5. **カテゴリの種類**
   - 命名規則: 変数名、関数名、クラス名等の命名に関する指摘
   - コードスタイル: インデント、空行、コメント等のスタイルに関する指摘
   - アーキテクチャ: Clean Architecture、DI、層間依存等の設計に関する指摘
   - 可読性: 関数の責務、ネストの深さ、重複コード等の可読性に関する指摘
   - エラーハンドリング: 例外処理、エラーメッセージ等に関する指摘
