## 出力ルール

1. **総合判定の決定**
   - いずれかのレビューが `NEEDS_FIX` → 総合判定も `NEEDS_FIX`
   - すべてのレビューが `APPROVED` → 総合判定も `APPROVED`
   - ただし、Critical な指摘がある場合は必ず `NEEDS_FIX`

2. **エグゼクティブサマリー**
   - 3-5文で簡潔にまとめる
   - 技術的な詳細は避け、全体像を伝える
   - ポジティブな面とネガティブな面をバランス良く記載

3. **レビューサマリー**
   - 各レビューの結果を簡潔にまとめる
   - 主要な指摘を1-2文で要約

4. **統合指摘事項**
   - 重複する指摘は統合（より厳しい重要度を採用）
   - 優先順位順（P0 → P1 → P2 → P3）に整理
   - 各指摘には「関連レビュー」を明記

5. **推奨アクション**
   - 具体的で実行可能なアクションを記載
   - 優先順位順に整理
   - 各アクションは1文で簡潔に

6. **次のステップ**
   - レビュー後に取るべき具体的なステップを記載
   - 例: 「P0 の2件を修正後、再レビューを依頼」
