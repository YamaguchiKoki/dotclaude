# コード品質レビュー

コードの品質レビューを実行してください。

## レビュー対象範囲

`git status` と `git diff` でレビュー対象ファイルを特定すること。
**変更されていない、レビュー対象外のファイルに対しレビューを行うことは固く禁じる。**

対象となる変更:
- `git diff`: unstaged な変更
- `git diff --staged`: staged な変更
- `git diff main...HEAD`: ブランチ全体の変更（必要に応じて）

## 実行手順

1. `git status` と `git diff` でレビュー対象ファイルを特定する
2. 対象ファイルに応じた適切なSkillを `CLAUDE.md` から参照し、コーディングルールを確認する
3. 以下のレビュー観点に基づき、各対象ファイルをレビューする
4. 指摘事項を `.tmp/review-results/code-quality.md` に出力する
5. `.tmp/review-results/code-quality.md` に出力されているかチェックする。**もし正しいパスに出力できていなければ出力をやり直す**

## レビュー観点

### 1. 命名規則チェック

`CLAUDE.md`及び命名規則のSkillsに準拠していること

### 2. コードスタイル

`CLAUDE.md`及びコーディンススタイルのSkillsに準拠していること

### 3. アーキテクチャ適合性

アーキテクチャ関連のSkillsに準拠していることをレビューする。

- **Clean Architecture層間依存**
  - API → Application → Domain → Infrastructure の依存方向が正しいか
  - 依存性逆転の原則に従っているか
- **DIコンテナ設定**
  - リポジトリのDI設定が適切に行われているか
  - インターフェース→実装のバインドが正しいか
- **ドメインロジックの配置**
  - ビジネスロジックがドメイン層に集約されているか
  - アプリケーション層で不適切なビジネスロジックがないか

### 4. 各Skillsごとにレビュー

各ファイルの実装内容をレビューする。`CLAUDE.md`に記載のSkillsから選択してレビュー基準として使用する。複数のSkillsを読み込んでも構わない。

## 各個のコメント形式

`pr-review-comment` skill の「インラインコメント」の項に従うこと。

## パイプライン出力

- **出力形式**: instruction で指定された出力形式に従うこと
- **出力先**: `.tmp/review-results/code-quality.md`
- **注意**: ローカルレビューのため、リモートへのコメント投稿は行わない
